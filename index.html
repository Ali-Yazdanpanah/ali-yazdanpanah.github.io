grep -rn "open questions" ./research

hardware & architecture

  • Which accelerator–memory co-design choices (banking, on-chip buffering, quantized datapaths) most reduce data movement per unit of useful work?
  • When does near-memory or in-memory computation genuinely remove data movement, rather than just relocating it?

gpu & parallel systems

  • How much GPU contention can be predicted from static, pre-execution signals, versus what genuinely requires runtime profiling?
  • Which memory-access and scheduling patterns dominate throughput for irregular, atomic-heavy GPU workloads?

distributed & cloud systems

  • How far can admission-time classification and lightweight runtime guards go toward safe multi-tenant accelerator sharing, without static hardware partitioning?
  • How should resource-arbitration decisions be split between a central orchestrator and node-local enforcement under strict SLAs?
